# Data_Analyst_Internship_2025

Day 1 – Data Cleaning & Preprocessing

Task Name: Data Cleaning and Preprocessing

Objective: To clean the raw dataset and make it structured, consistent, and ready for analysis.

Goals:

Identify and handle missing values
Remove duplicate records
Standardize inconsistent formats
Fix column names
Convert data types into correct formats
Prepare the final cleaned dataset

Tools Used:

Excel
(Optional) Python – Pandas

Dataset Used:

Customer Personality Analysis (Kaggle)

Steps Performed
1. Checking Missing Values
Used Excel filters to identify blank or null values and cleaned them.

2. Removing Duplicates
Used Excel → Data Tab → Remove Duplicates to remove repeated rows.

3. Text Standardization
Standardized gender values
Corrected inconsistencies in country/category names

4. Date Formatting
Converted all date fields into one consistent format: dd-mm-yyyy

5. Column Renaming
Cleaned the column names by making them lowercase and removing extra spaces.

6. Data Type Correction
Converted Age into integer format
Converted date columns into proper date/datetime format

Applied correct number formatting for numeric fields

Day 3 – SQL for Data Analysis
Task Name: SQL Queries for Data Extraction & Analysis
Objective

Extract meaningful insights from the database, join tables, and prepare structured analysis by filtering, sorting, and grouping data.

Goals:

Extract important columns
Apply filters using WHERE
Sort data using ORDER BY
Perform aggregation using GROUP BY
Join tables (INNER JOIN, LEFT JOIN)
Generate insights using SQL queries

Tools Used:
MySQL Workbench / MySQL CLI

Dataset Used
Customer Sales / Customer Personality Analysis
(Provided in the internship)

Steps Performed
1. Basic Data Extraction

Extracted key fields using SELECT statements
Applied conditions using WHERE to filter specific rows

2. Sorting the Data

Arranged records in ascending/descending order using ORDER BY
Identified high-value and low-value records efficiently

3. Data Aggregation

Used GROUP BY with functions like SUM(), COUNT(), AVG()
Performed country-wise, category-wise, and customer-type-wise analysis

4. Working with Joins

Used INNER JOIN to merge customer and transaction data
Used LEFT JOIN to identify missing or unmatched records
Generated combined insights using multi-table queries

5. Advanced SQL Filtering

Applied conditions using AND, OR, and numeric/text ranges
Performed date-based filtering such as year/month filters

6. Final Output Preparation

Generated clean and readable result sets
Documented all SQL queries and outputs in a Word file
Uploaded the SQL file and documentation in the Day 3 folder
